JAM - The Java API for Metadata

JAM is an object model for Java types and their associated annotations. It provides a superset of the features offered by other similar APIs (such as Reflection or X/Javadoc), including a unique extensibility mechanism specifically designed to help Java developers cope with the rapidly-changing world of Java metadata. About

JAM has evolved as part my work on XML-Beans at Apache. There, I needed to be able model Java types in order to generate XML schemas from them. I found all of the existing Java object models (javadoc, reflection, xdoclet) to be lacking in some respect. They all had features I wanted, but none of them had all of the features I wanted. And none of them provide a smooth upgrade path from javadoc-based annotations to JSR175, which is a big concern for me.

However, it is proving to be a generally useful technology in its own right. (Most notably, it is already being used in Cedric Beust's EJBgen and SGen). Accordingly, I am now spinning up JAM as a distinct open source project. Until that is done, my website here is serving to provide an early-access look at JAM.
